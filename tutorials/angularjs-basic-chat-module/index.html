You need PubNub API Keys. This allows the chat communication on a data stream network. You can fill in the YOUR_PUBLISH_API_KEY and YOUR_SUBSCRIBE_API_KEY placeholder strings with your API keys that you get on the PubNub website.

Now that you have the prerequisites and your PubNub API keys we are ready for the next part. Here's an example on how to send and receive a message within your AngularJS app. First we will start with accessing the AngularJS Chat Model methods. This code is a simple controller.

Controller for AngularJS Chat

{% highlight javascript %} // =-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-= // Minimal Chat Controller // =-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-= var chat = angular.module( 'BasicChat', ['chat'] ); chat.controller( 'chat', [ 'Messages', function( Messages ) { // - - - - - - - - - - - - - - - - - - // Message Inbox // This array will hold the messages // ordered by receipt. // - - - - - - - - - - - - - - - - - - this.messages = []; // - - - - - - - - - - - - - - - - - - // Receive Messages // Push to Message Inbox. // - - - - - - - - - - - - - - - - - - Messages.receive(function(message){ this.messages.push(message); }); // - - - - - - - - - - - - - - - - - - // Send Message // This is a controller method used // when a user action occurs. // Also we expect a model reference // ng-model="chat.textbox". // - - - - - - - - - - - - - - - - - - this.send = function() { Messages.send({ data : chat.textbox }); }; } ); {% endhighlight %}

You can put anything inside the object that is JSON serializable. This information will become accessible with each message sent. You can display this information in the HTML view. Internally we call this information chat message meta data. It will remain associated with each message sent. As of this tutorial, older messages will retain the original meta data information. This means if a user updates their name or info that it will not be propagated to the older chat messages.

One of the more difficult things to get right in application development is security. Security is included by default and for free. User communication security is paramount in chat applications. AES cryptography and TLS termination help secure encrypt user communication over the WAN. In addition to message body encryption, we also have to deal with message integrity and identity verification of the sender. How can you tell if the sender really is too they say they are. You needs to be old to confirm their identity. The best way to do that is through message signatures. You can also call these digital signatures LINK LINN LINK stackoverflow. This is done by creating and maintaining a secret on the client in sharing a public key that is maintained by in identity verification service. We had not included this feature yet. We will. Don't forget about identity authentication and authorization. When a user logs into their chat using their email and password that is called authenticating a user. After a user has been authenticated, they will receive a token known as a session ID. This session ID is a long unpredictable string representing randomized characters. This session ID will be used along with every REST Service call. Using the session ID is in a process called authorization. This is the phase where we check the session ID that has been validated in the authentication step. If the session ID is valid, we will look up the user permissions and allow them to pass returning a successful payload because they have been authorized. Even though it's the future and we have hover boards, Internet connectivity is often spotty and is absolutely unreliable in many situations. Especially when driving. Especially when walking around in the shopping mall. You need a reliable datastream network to provide message delivery. Even when your Internet disconnects your messages will still be delivered. This is called message recovery after disconnect and reconnect. Well there are many mechanisms to allow for message redelivery, we found at high scale networking becomes a big fun challenge. By default and currently the Best option we've included PubNub in AngularJS chat to solve this need. Long-term message storage requires is significantly different technical solution than short-term message redelivery. We've also use pub knob to help us solve for this. They have an infrastructure built with fault tolerance in mind. What they do is have 20+ data centers that maintain copies of your chat messages encrypted on a secure network. Angular JS chat Will make a REST call to the closest of the 20 data centers and receive historical message chat history. You may have noticed in the test application demo that message delivery is lightning fast. This is because we are using a network that is called PubNub. This network leverages many points of presence around the planet to deliver relay insecure message transfer in less than a quarter of a second.
